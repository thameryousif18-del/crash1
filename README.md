# بوت تلكرام لتحليل الأرقام (Signal Bot)

بوت تلكرام يستقبل أرقاماً (نتائج جولات) ويقارنها مع 3 شروط، ثم يرسل 🟢 أو 🔴.

## الشروط

1. آخر نتيجة بين 1.20 و 2.00
2. في آخر 7 جولات: تكرار الرقم 1.00 مرة واحدة كحد أقصى
3. آخر نتيجتين لا تحتويان أي رقم أقل من 1.02

إذا تحققت كل الشروط ← 🟢، وإذا فشل أي شرط ← 🔴.

## التنصيب

```bash
git clone <رابط_المستودع_الخاص_بك>
cd <اسم_المجلد>
pip install -r requirements.txt
```

## الإعداد

1. احصل على توكن البوت من [@BotFather](https://t.me/BotFather) على تلكرام.
2. انسخ ملف `.env.example` إلى `.env`:
   ```bash
   cp .env.example .env
   ```
3. افتح `.env` وضع التوكن الخاص بك:
   ```
   BOT_TOKEN=123456:ABC-your-real-token
   ```

## التشغيل

```bash
python signal_bot.py
```

## الأوامر داخل تلكرام

| الأمر | الوظيفة |
|---|---|
| `/start` | بدء البوت ومسح السجل |
| `/reset` | مسح سجل الأرقام والبدء من جديد |
| `/history` | عرض آخر الأرقام المسجلة |
| أي رقم (مثل `1.45`) | يُضاف للسجل ويُرجع النتيجة 🟢/🔴 |

## ملاحظات

- كل محادثة (Chat) لها سجل مستقل خاص بها (آخر 7 نتائج).
- **لا ترفع ملف `.env` الحقيقي إلى GitHub** — هو مستثنى تلقائياً عبر `.gitignore`. فقط `.env.example` يُرفع كنموذج.
- الشرط الثالث يحتاج نتيجتين على الأقل، لذا أول رقم يُرسل سيُصنَّف 🔴 تلقائياً لعدم اكتمال البيانات.

## النشر على سيرفر (اختياري)

لتشغيل البوت بشكل دائم (24/7) يمكنك استخدام أي VPS (مثل: Hostinger, DigitalOcean, Contabo) مع `screen` أو `systemd` أو `pm2`، أو منصات مثل Railway / Render.
